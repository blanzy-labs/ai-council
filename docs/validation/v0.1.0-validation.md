# AI Council v0.1.0 Validation Checklist

Use this checklist before tagging or publishing v0.1.0.

## Backend

- [ ] `GET /health` returns ok.
- [ ] `GET /version` returns `0.1.0`.
- [ ] `GET /personas` returns default personas.
- [ ] `POST /sessions` creates a session.
- [ ] `POST /sessions/{id}/run` works with mock provider.
- [ ] `POST /sessions/{id}/chat` works with mock council target.
- [ ] `POST /sessions/{id}/chat` works with mock persona target.
- [ ] `GET /sessions/{id}/events/recent` returns events.
- [ ] `POST /sessions/{id}/export` returns Markdown.
- [ ] `POST /sessions/{id}/export` returns JSON.
- [ ] `DELETE /sessions/{id}/messages` clears transcript.
- [ ] `DELETE /sessions/{id}/events` clears events.
- [ ] OpenAI missing-key response is clean.

## Frontend

- [ ] Health check displays.
- [ ] Personas display.
- [ ] Session creation works.
- [ ] Council run works with mock.
- [ ] Follow-up chat works with whole council.
- [ ] Follow-up chat works with one persona.
- [ ] Live activity displays.
- [ ] Markdown export displays/downloads.
- [ ] JSON export displays/downloads.
- [ ] Clear transcript works.
- [ ] Clear events works.
- [ ] Responsive layout is usable.

## Docker

- [ ] `docker compose up --build` succeeds.
- [ ] Frontend reachable at http://localhost:5173.
- [ ] Backend reachable at http://localhost:8000.

## Tests

- [ ] Backend tests pass.
- [ ] Frontend builds.

## Exact Validation Commands

Backend tests:

```sh
cd backend
uv run pytest
```

Frontend build:

```sh
cd frontend
pnpm build
```

Docker:

```sh
docker compose up --build
docker compose down
```

Backend metadata:

```sh
curl http://localhost:8000/health
curl http://localhost:8000/version
curl http://localhost:8000/personas
```

Mock session flow:

```sh
SESSION_JSON=$(curl -fsS -X POST http://localhost:8000/sessions \
  -H "Content-Type: application/json" \
  -d '{
    "title": "v0.1.0 validation",
    "topic": "Validate the AI Council release flow.",
    "mode": "ask_council",
    "selected_persona_ids": ["moderator", "skeptic", "builder"]
  }')

SESSION_ID=$(printf '%s' "$SESSION_JSON" | node -e "const fs=require('fs'); process.stdout.write(JSON.parse(fs.readFileSync(0, 'utf8')).id)")

curl -fsS -X POST "http://localhost:8000/sessions/$SESSION_ID/run" \
  -H "Content-Type: application/json" \
  -d '{
    "provider_override": "mock",
    "max_rounds": 1,
    "include_moderator_summary": true
  }'

curl -fsS -X POST "http://localhost:8000/sessions/$SESSION_ID/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What should we test next?",
    "target": {"type": "council"},
    "provider_override": "mock",
    "include_moderator_summary": false
  }'

curl -fsS -X POST "http://localhost:8000/sessions/$SESSION_ID/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Builder, what is the smallest useful next check?",
    "target": {"type": "persona", "persona_id": "builder"},
    "provider_override": "mock",
    "include_moderator_summary": false
  }'

curl -fsS "http://localhost:8000/sessions/$SESSION_ID/events/recent"

curl -fsS -X POST "http://localhost:8000/sessions/$SESSION_ID/export" \
  -H "Content-Type: application/json" \
  -d '{"format":"markdown","include_events":true,"include_metadata":true}'

curl -fsS -X POST "http://localhost:8000/sessions/$SESSION_ID/export" \
  -H "Content-Type: application/json" \
  -d '{"format":"json","include_events":true,"include_metadata":true}'

curl -fsS -X DELETE "http://localhost:8000/sessions/$SESSION_ID/messages"
curl -fsS -X DELETE "http://localhost:8000/sessions/$SESSION_ID/events"
```
